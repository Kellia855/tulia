TULIA  is  a  new  web-based  system  designed  to  support  early-stage  university  students  in  managing  
emotional

in

The  system  interacts  with  external  resources  such  as  helplines  and  other  mental  health  resources.  A  
high-level

TULIA’s
 
components

●  Web  application  interface :  for  student  interactions.  ●  Backend  server :  handles  data  storage,  processing  and  analytics.  ●  Database :  stores  student  inputs  securely  ensuring  privacy  and  confidentiality.  ●  External  resource  integration :  links  to  approved  mental  health  resources  and  helplines

2.2
 
 
Product  Functions  
At  a  high  level,  TULIA  enables  students  to:  
●  Log  and  track  their  daily  emotions  through  structured  check-ins   ●  Visualize  emotional  patterns  and  trends  over  time  using  a  dashboard  ●  Learn  and  expand  their  emotional  vocabulary  to  better  understand  and  express  their  feelings  ●  Reflect  on  personal  experiences  and  coping  strategies  ●  Connect  with  culturally  relevant  external  resources  and  mental  health  support

TULIA  is  designed  exclusively  for   early-stage  university  students,  providing  them  with  a  private,

3.1
 
 
User  Interfaces  
TULIA  provides  a  web-based  graphical  user  interface  (GUI)  designed  for  ease  of  use,  accessibility  and  
privacy.
 
Key
 
characteristics
 
include:
 
●  Consistent  layout:  Navigation  menus,  dashboards  and  feature  screens  follow  a  uniform  design   ●  Feature-specific  screens:                   1)          Emotional  check-ins:  simple  forms  to  log  emotions  with  optional  free-text  reflections  2)          Pattern  awareness  dashboard:  interactive  chart  and  graphs  to  visualize  emotional  trends  3)          Reflections:  text  input  area  linked  to  individual  check-ins  4)          Vocabulary  builder:  Interactive  exercises  with  selectable  answers  and  progress  tracking  5)          External  resources:  links  displayed  clearly  with  descriptions  and  accessibility  labels  ●  Error  handling:  Clear  error  messages  for  invalid  inputs,  connectivity  issues,  etc  ●  Accessibility:  The  interface  follows  web  accessibility  guidelines  (WCAG)  for  readability  and  
usability

Requirement  Specification   
 
Functional  Requirements   
Req  ID  Requirements  Description   
FR  1  User  authentication  The  system  shall  allow  university  students  to  register  and  securely  authenticate  using  login  credentials  
FR  2  Emotional  check-ins  The  system  shall  enable  users  to  record  emotional  check-ins  from  a  structured  emotional  vocabulary  
FR  3  Reflection  management  The  system  shall  allow  users  to  create,  edit,view  and  delete  personal  emotions  reflections  
FR  4  Emotional  vocabulary  learning  
The  system  shall  provide  definitions  and  examples  of  emotions  to  help  users  improve  emotional  literacy  
FR  5  Pattern  awareness  dashboard  The  system  shall  analyze  check-in  data  and  display  visual  summaries  of  emotional  pattern  over  time  
FR  6  External  support  resources  The  system  shall  provide  access  to  curated  mental  health  resources  and  helplines  to  university  students  in  Africa   
FR  7  Data  storage  and  retrieval  The  system  shall  securely  store  and  retrieve  user  emotional  data  including  check-ins  and  reflections  
FR  8  User  data  deletion  The  system  shall  allow  users  to  permanently  delete  their  account  and  associated  emotional  data  upon  request
